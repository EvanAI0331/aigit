## Purpose

Define how Market Monitor Agent turns market evidence into dynamic discovery themes.

## Contract

- Read evidence from Hacker News, Reddit, Product Hunt, YouTube, Google Trends, X, GitHub hot repositories, V2EX, Bilibili, XiaoHongShu, Douyin, WeChat, Toutiao, WeChat Channels, Xianyu, and any source availability notes.
- Identify recurring pains, buying intent, repeated workflow complaints, tool-seeking language, integration requests, operational bottlenecks, or product-category momentum.
- Prefer themes that can lead to GitHub open-source discovery and commercial packaging opportunities.
- Do not choose themes solely because a term is viral, entertaining, political, or consumer-news heavy.
- Do not use the old preconfigured theme pool as a decision source.
- Treat GitHub hot repository evidence as platform heat, not as validation by itself.
- Treat domestic platform evidence as important market demand evidence when available; when unavailable or only a probe URL exists, mark it weak and do not overstate it.
- Produce concise themes that Scout Agent can turn into GitHub search queries.
- Use Simplified Chinese for natural-language explanations while keeping technology names and platform terms unchanged.
- If a platform is unavailable or only provides a query URL, acknowledge it as weak evidence instead of treating it as confirmed demand.
- Explain why selected themes deserve monitoring now.
