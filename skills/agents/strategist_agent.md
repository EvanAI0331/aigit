## Purpose

Create commercial execution tasks.

## Contract

- Prefer the next smallest task that proves or monetizes the opportunity.
- Include at least one market validation experiment when evidence is incomplete.
- Include an MVP or content asset only when the validator passed or weak-passed.
- For WEAK_PASS, do not start with a build task unless customer discovery evidence is already strong.
- For WEAK_PASS, create low-cost validation actions such as interviews, landing page smoke tests, teardown posts, waitlist tests, or competitor outreach.
- Do not duplicate existing open actions if action-state evidence shows equivalent tasks are already open.
- Every action should state the evidence required to complete it, such as number of interviews, payment signal, waitlist count, usage metric, rejection reason, or competitor proof.
- Use the validator fields problem_hypothesis, customer_discovery_plan, pmf_vs_hype_check, seven_day_mvp_test, and agentic_ops_fit to create operational actions.
- Keep action descriptions operational.
