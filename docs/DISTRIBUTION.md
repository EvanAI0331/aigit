# Distribution Checklist

## GitHub

- [ ] Keep description under 160 characters.
- [ ] Add topics: `ai-agents`, `github-analytics`, `startup-tools`, `opensource-intelligence`, `llmops`, `agentic-ai`, `founder-tools`.
- [ ] Pin the repo on the GitHub profile.
- [ ] Create a first release with a clear founder-oriented changelog.
- [ ] Add screenshots or a short demo GIF when the frontend is wired to live API data.
- [ ] Convert common questions into issues labeled `good first issue`, `collector`, `agent-spec`, and `frontend`.

## Launch Sequence

1. GitHub release.
2. X / Twitter short launch thread.
3. LinkedIn long post.
4. Hacker News Show HN.
5. Reddit side-project and open-source posts.
6. Product Hunt launch after a demo screenshot or short video is ready.
7. Follow-up technical article about spec/skill-driven agents.

## Message Angles

- For founders: discover software business opportunities from open-source momentum.
- For agent builders: strict harness with specs, skills, contracts, and fail-closed runtime.
- For product teams: turn GitHub activity into validated market signals.
- For open-source maintainers: learn which repos may support commercial packaging.

## Weekly Growth Loop

- Publish one validated opportunity teardown per week.
- Open issues for missing collectors and validation gates.
- Share benchmark runs and rejected examples, not only successes.
- Invite contributors to add agent skills, source collectors, and frontend views.
- Keep the README first screen focused on the value proposition and quick start.
